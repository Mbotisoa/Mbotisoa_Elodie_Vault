### Publication du Vault

*Le Vault Obsidian* a été publié en ligne à l’aide de la plateforme GitHub afin d’assurer l’accessibilité, l’archivage et la traçabilité du projet. Un dépôt dédié a été créé pour héberger l’ensemble des fichiers du projet, incluant les notes Obsidian, les ressources graphiques et les fichiers web.

### Procédure de publication

*La procédure de publication* repose sur l’utilisation de Git pour le versioning. Les fichiers du projet sont ajoutés progressivement au dépôt, et chaque étape importante fait l’objet d’un commit documenté. Cette méthode permet de conserver un historique clair de l’évolution du projet et de revenir sur des versions antérieures si nécessaire.

Un fichier README accompagne le dépôt afin de présenter le projet,  les outils utilisés et la structure générale du Vault.

### Accessibilité et archivage

*La publication sur GitHub* permet de rendre le projet accessible sans dépendre exclusivement d’Obsidian, notamment via l’export des contenus ou l’utilisation de GitHub Pages pour une consultation web. Le dépôt GitHub constitue également une solution d’archivage pérenne, garantissant la conservation du projet au-delà du cadre du cours.
